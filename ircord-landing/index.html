architecture

Simple enough to understand, modern enough to matter

IRCord separates client, transport and trust boundaries cleanly. The server coordinates delivery and presence, while encrypted message content stays on the client side.

client Desktop / CLI Fast terminal-style interaction, command shortcuts, room navigation, local encryption and voice controls.
->
transport IRCord Server Presence, room membership, routing, signaling, file metadata and session coordination.
->
trust boundary Encrypted Payloads Messages are encrypted before transport. Server operators get metadata and routing, not your plaintext conversations.

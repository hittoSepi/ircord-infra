IRCord Server

Run your own encrypted relay server. The server never sees your messages — it just routes encrypted data between clients.

What is IRCord Server

IRCord Server is a lightweight relay built with C++20 and Boost.Asio. It accepts TLS/TCP connections, speaks a length-prefixed Protobuf wire protocol, and forwards encrypted payloads between clients. The server itself has no ability to decrypt message content — all cryptographic operations happen on the client side using the Signal Protocol for DMs and Sender Keys for channels.

User accounts and public key bundles are persisted in a local SQLite database. The server handles authentication, channel state, user presence, and offline message queuing, but it never touches plaintext.

Getting Started

Pre-built binaries are available for Linux aarch64 (Raspberry Pi) and Linux x64. Download the latest release, place the binary somewhere convenient, and create a server.toml configuration file.

bash 
# Download and extract
tar xzf ircord-server-linux-aarch64.tar.gz
cd ircord-server

# Create a minimal config (see Configuration section below)
cp server.example.toml server.toml

# Start the server
./ircord-server --config server.toml
Check your build

Run ./ircord-server --version to see the git commit hash and build timestamp. This is useful for verifying that the correct version is deployed, especially on remote machines.

By default the server listens on port 6697 (the standard IRC-over-TLS port). You will need a valid TLS certificate and private key — see the Infrastructure guide for details on obtaining certificates with Let's Encrypt.

Configuration Reference

The server is configured through a single server.toml file. Below is a complete reference of all available options.

[server]

Key Type Default Description
host string "0.0.0.0" Bind address. Use "0.0.0.0" to listen on all interfaces.
port integer 6697 TCP port for TLS connections.
max_connections integer 256 Maximum simultaneous client connections.
motd string "" Message of the Day, shown to clients on connect.

[tls]

Key Type Default Description
cert_file string — Path to the TLS certificate file (PEM format). Required.
key_file string — Path to the TLS private key file (PEM format). Required.

[database]

Key Type Default Description
path string "ircord.db" Path to the SQLite database file. Created automatically if it does not exist.

[directory]

Key Type Default Description
enabled boolean false Register this server with the public directory so others can discover it.
server_name string "" Display name shown in the directory listing.
description string "" Short description of the server.
directory_url string "" URL of the directory server to register with.

[limits]

Key Type Default Description
message_rate integer 20 Maximum chat messages per second per user.
command_rate integer 30 Maximum commands per minute per user.
join_rate integer 5 Maximum channel joins per minute per user.
connect_rate integer 10 Maximum new connections per minute per IP address.

[offline]

Key Type Default Description
enabled boolean true Store messages for offline users and deliver them on reconnect.
ttl_days integer 7 Number of days to retain offline messages before they expire.
max_per_user integer 100 Maximum offline messages stored per recipient.
max_total integer 100000 Maximum total offline messages across all users.

Example Configuration

toml 
[server]
host = "0.0.0.0"
port = 6697
max_connections = 256
motd = "Welcome to my IRCord server!"

[tls]
cert_file = "/etc/letsencrypt/live/example.com/fullchain.pem"
key_file  = "/etc/letsencrypt/live/example.com/privkey.pem"

[database]
path = "/var/lib/ircord/ircord.db"

[directory]
enabled      = true
server_name  = "My IRCord"
description  = "A friendly encrypted chat server"
directory_url = "https://directory.ircord.org"

[limits]
message_rate = 20
command_rate = 30
join_rate    = 5
connect_rate = 10

[offline]
enabled      = true
ttl_days     = 7
max_per_user = 100
max_total    = 100000

IRC Commands

IRCord supports a familiar set of IRC-style slash commands. Commands are processed by the server (except for client-local commands like /quit).

Command Description Example
/join Join a channel. Creates it if it does not exist. /join #general
/part Leave the current channel. /part #general
/nick Change your display name. /nick newname
/msg Send a private (encrypted) direct message. /msg alice Hey there
/me Send an action message (third person). /me waves hello
/whois View information about a user. /whois alice
/quit Disconnect from the server. /quit Goodbye!
/password Change your account password. /password oldpass newpass
/kick Remove a user from the channel (operator only). /kick bob Spamming
/ban Ban a user from the channel (operator only). /ban bob
/invite Invite a user to an invite-only channel. /invite alice #secret
/topic View or set the channel topic. /topic #general New topic text
/set Change a channel option (operator only). /set invite_only true

Channel Management

Channels in IRCord follow the IRC convention of starting with #. The first user to join a channel automatically becomes its operator and can manage settings and moderate users.

User Modes

Prefix Role Capabilities
@ Operator Kick, ban, invite, set topic, change channel options, grant/revoke modes.
+ Voiced Can speak in moderated channels. No moderation powers.
(none) Regular Can read and send messages in normal channels. Cannot speak in moderated channels.

Channel Options

Operators can configure channel behavior with the /set command.

Option Type Default Description
invite_only boolean false Only invited users may join. Use /invite to add users.
moderated boolean false Only operators and voiced users can send messages.
topic_restricted boolean false Only operators can change the channel topic.
irc 
/join #private
/set invite_only true
/set topic_restricted true
/topic #private Members-only discussion
/invite alice #private

Rate Limiting

The server enforces rate limits to prevent abuse and ensure fair resource usage. These limits apply per user (or per IP for connection limits) and are configurable in server.toml.

Action Default Limit Window
Chat messages 20 Per second
Commands 30 Per minute
Channel joins 5 Per minute
Connections (per IP) 10 Per minute

When a user exceeds a rate limit, the server drops the offending message and sends a warning. After 5 violations within a session, the user is automatically banned for 30 minutes.

Automated clients and bots

If you are developing a plugin or bot, be mindful of rate limits. Sending messages in tight loops will trigger the auto-ban. Use delays between messages, and batch operations where possible. See the Plugin Development guide for best practices.

Offline Messages

When a recipient is not connected, the server stores their incoming messages and delivers them automatically when they reconnect. This applies to both direct messages and channel messages directed at offline users.

Default limits for offline message storage:

  • TTL: 7 days — messages older than this are automatically purged.
  • Per user: 100 messages maximum per recipient.
  • Total: 100,000 messages across all users on the server.

When limits are exceeded, the oldest messages are discarded first. All limits are configurable in the [offline] section of server.toml.

End-to-end encryption preserved

Offline messages are stored in their encrypted form. The server cannot read them — it simply holds the ciphertext until the recipient comes online. Your messages remain private even while queued on the server.

Version Tracking

The server embeds its git commit hash and build timestamp at compile time. You can check the version of any running server using the --version flag:

bash 
$ ./ircord-server --version
ircord-server v0.15.0 (commit a3f7c2e, built 2026-03-13T14:22:01Z)

The version is also logged on startup, making it easy to verify the correct binary is running on remote machines by checking the server logs:

log 
[2026-03-13 14:22:05] [info] ircord-server v0.15.0 (commit a3f7c2e)
[2026-03-13 14:22:05] [info] Listening on 0.0.0.0:6697 (TLS)
[2026-03-13 14:22:05] [info] Database: /var/lib/ircord/ircord.db
[2026-03-13 14:22:05] [info] Ready.
Deployment verification

After deploying a new build to a remote server, SSH in and run ./ircord-server --version to confirm the update took effect. Compare the commit hash against your local build to ensure they match.

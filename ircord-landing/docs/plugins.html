Plugin Development Guide

Build bots, client extensions, and server extensions for IRCord using JavaScript (ES2023). Plugins run in a sandboxed QuickJS runtime.

Introduction

The IRCord plugin system lets you extend every layer of the platform. There are three types of plugins, each with different capabilities and constraints:

  • Bot plugins — standalone programs that connect to the server as their own user and can participate in channels, respond to commands, and integrate with external services.
  • Client extensions — run inside a user's client process, adding local commands, UI tweaks, and custom behavior without requiring a separate connection.
  • Server extensions — run inside the server process and handle administrative tasks like channel management, rate limiting, and audit logging. Because of end-to-end encryption, server extensions never see message content.

All plugins are written in JavaScript (ES2023) and execute in a sandboxed QuickJS runtime. Each plugin gets its own isolated heap with strict resource limits, so a misbehaving plugin cannot crash the host or interfere with other plugins.

E2E Encryption Principle

The IRCord server is a relay — it never decrypts messages. This means server extensions can only access metadata (who sent a message, to which channel, timestamps), never the actual content. Bot plugins and client extensions can see content because they operate on the client side of the encryption boundary.

Quick Start: Your First Bot

This walkthrough creates a complete DiceBot that responds to /roll commands. By the end you will have a working bot that rolls dice and reports results in any channel it joins.

Step 1: Create the Plugin Directory

Create a folder for your plugin. The bot host scans subdirectories of its plugins path for plugin.json files.

bash 
mkdir -p ~/.config/ircord/plugins/dicebot/

Step 2: Create plugin.json

Every plugin requires a plugin.json manifest. This tells the host what to load, what permissions the plugin needs, and what commands it registers.

json 
{
  "name": "dicebot",
  "display_name": "Dice Bot",
  "version": "1.0.0",
  "type": "bot",
  "entry": "main.js",
  "description": "Roll dice with /roll command",
  "min_host_version": "0.1.0",
  "bot": {
    "user_id": "DiceBot",
    "auto_join_channels": ["#general"]
  },
  "commands": [
    {
      "name": "/roll",
      "description": "Roll dice (e.g. /roll 2d6)",
      "usage": "/roll [NdN | N]"
    }
  ],
  "permissions": ["read_messages", "send_messages", "register_commands"]
}

Step 3: Create main.js

The entry point script. The ircord global object provides all the APIs you need to interact with the server and handle commands.

javascript 
// DiceBot — simple dice roller

function parseDice(input) {
    if (!input) return { count: 1, sides: 6 };
    var match = input.match(/^(\d+)d(\d+)$/i);
    if (!match) {
        var n = parseInt(input);
        if (!isNaN(n) && n >= 1 && n <= 100) return { count: 1, sides: n };
        return null;
    }
    var count = parseInt(match[1]);
    var sides = parseInt(match[2]);
    if (count < 1 || count > 20 || sides < 2 || sides > 1000) return null;
    return { count: count, sides: sides };
}

function rollDice(count, sides) {
    var rolls = [];
    for (var i = 0; i < count; i++) {
        rolls.push(Math.floor(Math.random() * sides) + 1);
    }
    return rolls;
}

ircord.onCommand("/roll", {
    description: "Roll NdN dice",
    usage: "/roll [NdN]"
}, function(ctx) {
    var dice = parseDice(ctx.args[0]);
    if (!dice) {
        ircord.sendMessage(ctx.channel,
            ctx.sender + ": Invalid format. Usage: /roll 2d6");
        return;
    }
    var rolls = rollDice(dice.count, dice.sides);
    var total = 0;
    for (var i = 0; i < rolls.length; i++) total += rolls[i];
    if (dice.count === 1) {
        ircord.sendMessage(ctx.channel,
            ctx.sender + ": d" + dice.sides + " = " + total);
    } else {
        ircord.sendMessage(ctx.channel,
            ctx.sender + ": " + dice.count + "d" + dice.sides
            + " = " + rolls.join(" + ") + " = " + total);
    }
});

ircord.log.info("DiceBot loaded!");

Step 4: Launch the Bot Host

Start the bot host process, pointing it at your plugins directory. The bot host manages connections, encryption, and plugin lifecycles.

bash 
ircord-bot-host --config bot-host.toml --plugins ./plugins/
Hot Reload During Development

Use hot reload during development — edit your JS and the bot reloads without disconnecting. See the Hot Reload section for details.

Plugin Types

The three plugin types serve different purposes and operate at different layers. Choose the type that matches your use case.

Feature Bot Client Extension Server Extension
Runs in Own process User's client Server
Has network connection Yes (own) No (uses host) No (uses host)
Sees message content Yes Yes No (E2E)
Can send messages Yes Yes No
Can manage channels No No Yes
Can kick/ban users No No Yes
Active when Bot host runs User is online Server runs

Bot Plugins

Bots connect to the server as their own user identity. They are ideal for interactive commands, FAQ bots, and integrations with external services like weather APIs, RSS feeds, or CI/CD pipelines. A bot participates in channels just like a regular user and can read and send encrypted messages.

Client Extensions

Client extensions run inside the user's own client process. They can add local slash commands (like /clear or /theme), modify how messages are displayed, add syntax highlighting, or trigger desktop notifications. Client extensions do not create additional server connections.

Server Extensions

Server extensions run inside the server process and have access to administrative functions: creating and deleting channels, kicking and banning users, rate limiting, and audit logging. Because IRCord uses end-to-end encryption, server extensions only see metadata — they cannot read message content.

plugin.json Reference

The plugin.json manifest is the only required configuration file. It tells the plugin host how to load, identify, and sandbox your plugin.

Field Type Required Description
name string Yes Unique plugin identifier, lowercase alphanumeric and hyphens only.
display_name string No Human-readable name shown in UI and logs.
version string Yes Semantic version (e.g. "1.0.0").
type string Yes One of "bot", "client_extension", or "server_extension".
entry string Yes Path to the main JavaScript file, relative to the plugin directory.
description string No Short description of what the plugin does.
min_host_version string No Minimum required version of the host (bot host, client, or server).
bot object No Bot-specific settings. Only used when type is "bot".
bot.user_id string No The username the bot will use when connecting to the server.
bot.auto_join_channels string[] No List of channels to join automatically on startup.
commands array No List of slash commands the plugin registers.
commands[].name string No Command name including the leading slash (e.g. "/roll").
commands[].description string No Short description shown in help/autocomplete.
commands[].usage string No Usage syntax (e.g. "/roll [NdN]").
permissions string[] No List of permissions the plugin requires. See Permissions.

JavaScript API: Common

The ircord global object is injected into every plugin's runtime. The following functions are available to all plugin types.

ircord.on(event_name, callback)

Register a handler for a named event. See Events for the full list.

javascript 
ircord.on("message", function(event) {
    ircord.log.info("Message from " + event.sender + ": " + event.text);
});

ircord.onCommand(name, options, callback)

Register a slash command. The options object accepts description and usage strings. The callback receives a context object with channel, sender, and args (an array of arguments).

javascript 
ircord.onCommand("/ping", {
    description: "Check if the bot is alive",
    usage: "/ping"
}, function(ctx) {
    ircord.sendMessage(ctx.channel, ctx.sender + ": Pong!");
});

ircord.sendMessage(channel, text)

Send a message to a channel or user. For channels, prefix with #. For direct messages, use the user ID.

javascript 
ircord.sendMessage("#general", "Hello, world!");
ircord.sendMessage("alice", "This is a direct message.");

ircord.getConfig()

Returns the contents of the plugin's config.toml file as a JavaScript object. Returns an empty object if no config file exists.

javascript 
var config = ircord.getConfig();
var apiKey = config.api_key || "default-key";

ircord.getPluginDir()

Returns the absolute path to the plugin's root directory.

javascript 
var dir = ircord.getPluginDir();
ircord.log.info("Running from: " + dir);

ircord.getDataDir()

Returns the absolute path to the plugin's data/ directory. This is the only location where file I/O is allowed.

javascript 
var dataDir = ircord.getDataDir();
// dataDir === "/home/user/.config/ircord/plugins/mybot/data/"

ircord.log.info / warn / error / debug(msg)

Structured logging with severity levels. Log output is tagged with the plugin name and written to the host's log stream.

javascript 
ircord.log.info("Bot started successfully");
ircord.log.warn("Config file missing, using defaults");
ircord.log.error("Failed to fetch API data");
ircord.log.debug("Received event: " + JSON.stringify(event));

ircord.setTimeout / setInterval / clearTimeout / clearInterval

Standard timer functions. Intervals and timeouts are automatically cleaned up when a plugin is unloaded.

javascript 
var timerId = ircord.setInterval(function() {
    ircord.sendMessage("#general", "Hourly reminder: stay hydrated!");
}, 3600000);

// Cancel when no longer needed
ircord.clearInterval(timerId);

ircord.http.get(url, options) / ircord.http.post(url, body, options)

Make HTTP requests. Requires the http_request permission. Returns a Promise that resolves with { status, body, headers }. Subject to rate limiting and timeout restrictions (see Sandbox).

javascript 
ircord.http.get("https://api.example.com/data", {
    headers: { "Authorization": "Bearer " + token }
}).then(function(res) {
    var data = JSON.parse(res.body);
    ircord.log.info("Got " + data.items.length + " items");
}).catch(function(err) {
    ircord.log.error("HTTP request failed: " + err.message);
});

ircord.http.post("https://api.example.com/webhook",
    JSON.stringify({ text: "Hello" }),
    { headers: { "Content-Type": "application/json" } }
);

ircord.fs.readFile(path) / ircord.fs.writeFile(path, content)

Read and write files within the plugin's data/ directory. Paths are relative to data/. Requires file_read or file_write permission respectively. Files are limited to 1 MB.

javascript 
// Save state
var state = { lastRun: Date.now(), counter: 42 };
ircord.fs.writeFile("state.json", JSON.stringify(state));

// Load state
var raw = ircord.fs.readFile("state.json");
if (raw) {
    var loaded = JSON.parse(raw);
    ircord.log.info("Counter: " + loaded.counter);
}

JavaScript API: Bot

The following functions are available only to bot plugins (type "bot"). They provide control over the bot's identity and presence on the server.

ircord.bot.userId

Read-only property containing the bot's user ID string, as configured in plugin.json.

javascript 
ircord.log.info("Running as: " + ircord.bot.userId);

ircord.bot.joinChannel(channel)

Join a channel at runtime. The bot will begin receiving messages from the channel.

javascript 
ircord.bot.joinChannel("#support");

ircord.bot.leaveChannel(channel)

Leave a channel. The bot will stop receiving messages from the channel.

javascript 
ircord.bot.leaveChannel("#old-channel");

ircord.bot.setPresence(status)

Set the bot's presence status. Valid values are "online" and "away".

javascript 
// Go away during maintenance
ircord.bot.setPresence("away");

// Come back
ircord.bot.setPresence("online");

JavaScript API: Client Extension

The following functions are available only to client extensions (type "client_extension"). They provide read access to the client's current state and the ability to show notifications.

ircord.client.activeChannel()

Returns the name of the channel the user currently has focused.

javascript 
var channel = ircord.client.activeChannel();
ircord.log.info("User is viewing: " + channel);

ircord.client.onlineUsers()

Returns an array of user ID strings for all currently online users.

javascript 
var users = ircord.client.onlineUsers();
ircord.log.info(users.length + " users online");

ircord.client.channels()

Returns an array of channel name strings the user has joined.

javascript 
var channels = ircord.client.channels();
// ["#general", "#dev", "#random"]

ircord.client.notify(text)

Display a notification in the client's status bar. Useful for transient alerts that should not be stored in chat history.

javascript 
ircord.client.notify("Theme changed to dark mode");

JavaScript API: Server Extension

The following functions are available only to server extensions (type "server_extension"). These provide administrative control over the server.

No Message Content Access

Server extensions never have access to message content. This is enforced at the protocol level — the server only relays encrypted payloads. Server extension event handlers receive metadata (sender, channel, timestamp) but the text field is always empty.

ircord.server.kickUser(user_id, reason)

Disconnect a user from the server with an optional reason message.

javascript 
ircord.server.kickUser("spammer123", "Flooding the channel");

ircord.server.banUser(user_id, reason)

Ban a user from the server. Banned users cannot reconnect until the ban is lifted.

javascript 
ircord.server.banUser("spammer123", "Repeated violations");

ircord.server.createChannel(channel_id)

Create a new channel on the server. Channel IDs must start with #.

javascript 
ircord.server.createChannel("#project-alpha");

ircord.server.deleteChannel(channel_id)

Delete a channel from the server. All members are removed.

javascript 
ircord.server.deleteChannel("#archived-project");

ircord.server.onlineUsers()

Returns an array of user ID strings for all currently connected users.

javascript 
var users = ircord.server.onlineUsers();
ircord.log.info("Connected users: " + users.length);

ircord.server.channelMembers(channel_id)

Returns an array of user ID strings for all members of a given channel.

javascript 
var members = ircord.server.channelMembers("#general");
ircord.log.info("#general has " + members.length + " members");

Permissions

Plugins must declare the permissions they require in plugin.json. The host will refuse to load a plugin that uses APIs it has not declared permissions for. Not all permissions are available to all plugin types.

Permission Bot Client Ext Server Ext Description
read_messages Yes Yes No Receive and read message content.
send_messages Yes Yes No Send messages to channels and users.
register_commands Yes Yes No Register slash commands.
manage_channels No No Yes Create and delete channels.
manage_users No No Yes Kick and ban users.
http_request Yes Yes Yes Make outbound HTTP requests.
file_read Yes Yes Yes Read files from the plugin's data directory.
file_write Yes Yes Yes Write files to the plugin's data directory.

Events

Events are the primary way plugins receive information from the host. Use ircord.on(event_name, callback) to register handlers. A plugin can register multiple handlers for the same event.

Event Available To Description
message Bot, Client Ext A message was sent in a channel the plugin is watching.
user_joined All A user joined a channel.
user_left All A user left a channel.
presence_changed All A user's presence status changed (online/away).
channel_created All A new channel was created on the server.
channel_deleted All A channel was deleted from the server.
connection_established Server Ext A new client connected to the server.
connection_dropped Server Ext A client disconnected from the server.
voice_room_joined All A user joined a voice room.
voice_room_left All A user left a voice room.
auth_success Server Ext A user successfully authenticated.
auth_failure Server Ext An authentication attempt failed.

Event Callback Object

Every event callback receives a single object with the following fields. Not all fields are populated for every event type.

Field Type Description
type string The event type name (e.g. "message", "user_joined").
channel string The channel where the event occurred, if applicable.
sender string The user ID of the user who triggered the event.
text string Message content (only for message events, empty in server extensions).
timestamp number Unix timestamp in milliseconds.
javascript 
ircord.on("message", function(event) {
    ircord.log.info("[" + event.channel + "] " + event.sender + ": " + event.text);
});

ircord.on("user_joined", function(event) {
    ircord.log.info(event.sender + " joined " + event.channel
        + " at " + new Date(event.timestamp).toISOString());
});

ircord.on("presence_changed", function(event) {
    ircord.log.info(event.sender + " is now " + event.text);
});

File Structure

A plugin directory follows this layout. Only plugin.json and the entry point script are required; everything else is optional.

text 
my-plugin/
├── plugin.json        — required: metadata and configuration
├── main.js            — required: entry point script
├── config.toml        — optional: plugin configuration file
├── lib/               — optional: additional JS modules
│   └── utils.js
└── data/              — runtime: persistent plugin data (auto-created)
    └── state.json

Key points about the file structure:

  • plugin.json must be at the root of the plugin directory. The host scans for this file to discover plugins.
  • Entry point (usually main.js) is loaded and executed when the plugin starts. The filename is configured via the entry field in plugin.json.
  • config.toml is parsed and made available via ircord.getConfig(). Use it for API keys, thresholds, and other tunable settings so users can customize behavior without editing code.
  • lib/ contains additional JavaScript modules. Use require("./lib/utils.js") to load them.
  • data/ is the only directory where ircord.fs can read and write files. It is created automatically on first write. File I/O is sandboxed to this directory — attempts to access paths outside it will fail.

Hot Reload

During development, you can edit your plugin's JavaScript files and the host will automatically reload the plugin without dropping the bot's server connection. This provides a fast edit-test cycle.

How hot reload works:

  1. You save a change to any .js file in the plugin directory.
  2. The plugin manager detects the file change via filesystem watch.
  3. The current plugin instance's onShutdown handler is called (if registered).
  4. The QuickJS runtime is destroyed and a fresh one is created.
  5. The entry point script is re-executed in the new runtime.
  6. All event handlers and commands are re-registered from scratch.

The bot's connection to the server stays alive throughout this process. Channel memberships, the bot's user identity, and the data/ directory are all preserved.

javascript 
// Handle graceful shutdown (called before hot reload or unload)
ircord.on("shutdown", function() {
    // Save any in-memory state before reload
    var state = { counter: messageCount, lastActive: Date.now() };
    ircord.fs.writeFile("state.json", JSON.stringify(state));
    ircord.log.info("State saved before reload");
});

// Restore state on startup
var saved = ircord.fs.readFile("state.json");
var messageCount = 0;
if (saved) {
    var state = JSON.parse(saved);
    messageCount = state.counter || 0;
    ircord.log.info("Restored counter: " + messageCount);
}
Development Workflow

Keep a terminal open with the bot host running and your editor side by side. Every time you save, the plugin reloads instantly. Check the bot host's log output for any errors during reload.

Sandbox & Limits

Each plugin runs in an isolated QuickJS runtime with strict resource limits. These limits protect the host process and other plugins from resource exhaustion or runaway code.

Resource Limit
Memory (JS heap) 16 MB per plugin
Stack size 1 MB
Max errors per hour 50 (auto-unload on exceed)
HTTP request timeout 10 seconds
HTTP rate limit 30 requests per minute
File size limit 1 MB per file
Automatic Unloading

Exceeding the error limit (50 errors per hour) will cause your plugin to be automatically unloaded. The host logs the reason for unloading. Fix the errors and reload the plugin manually, or restart the host to retry.

Additional sandbox restrictions:

  • No access to the host's filesystem outside the plugin's data/ directory.
  • No access to environment variables or system commands.
  • No raw socket access — only the provided HTTP client is available for network calls.
  • No spawning child processes.
  • No access to other plugins' runtimes or data.
  • The eval() function is disabled.

More Examples

WelcomeBot

A simple bot that greets users when they join a channel.

javascript 
ircord.on("user_joined", function(event) {
    ircord.sendMessage(event.channel,
        "Welcome to " + event.channel + ", " + event.sender + "!");
});

ircord.log.info("WelcomeBot loaded!");

Channel Logger (Server Extension)

A server extension that logs join and leave events to an audit file. Because this is a server extension, it only sees metadata — no message content.

javascript 
ircord.on("user_joined", function(event) {
    var line = new Date(event.timestamp).toISOString()
        + " JOIN " + event.sender + " " + event.channel + "\n";
    ircord.fs.writeFile("audit.log", line);
});

ircord.on("user_left", function(event) {
    var line = new Date(event.timestamp).toISOString()
        + " PART " + event.sender + " " + event.channel + "\n";
    ircord.fs.writeFile("audit.log", line);
});

ircord.log.info("Channel Logger loaded!");

Weather Bot

A bot that fetches current weather data from an external API. Requires the http_request permission.

javascript 
ircord.onCommand("/weather", {
    description: "Get current weather",
    usage: "/weather [city]"
}, function(ctx) {
    var city = ctx.args.join(" ") || "Helsinki";
    ircord.http.get("https://wttr.in/" + encodeURIComponent(city) + "?format=3")
        .then(function(res) {
            ircord.sendMessage(ctx.channel, res.body);
        })
        .catch(function(err) {
            ircord.sendMessage(ctx.channel, "Could not fetch weather: " + err.message);
        });
});

ircord.log.info("WeatherBot loaded!");
